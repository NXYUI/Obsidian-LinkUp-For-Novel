# AIOFN

AIOFN은 소설 집필 과정에서 흩어지기 쉬운 자료를 하나의 Obsidian 볼트 안에서 정리하고, 그 자료를 더 쉽게 읽고 연결하고 점검하기 위한 데스크톱 앱입니다.

이 앱은 원고를 대신 써주는 도구라기보다, 작가가 이미 만들어 둔 세계관, 인물, 설정, 플롯, 회차 메모를 작업 가능한 형태로 다시 보여주는 `보조 작업대`에 가깝습니다.

Obsidian의 자유로운 문서 구조는 그대로 유지하면서, 장편 작업에서 반복적으로 필요한 정리, 확인, 이동, 백업 과정을 줄이는 것을 목표로 합니다.

특히 문서가 많아질수록 생기는 문제에 초점을 둡니다.

어떤 설정이 어디에 있었는지 찾기 어렵거나, 캐릭터와 사건의 연결이 흐려지거나, 이전에 적어 둔 내용과 새로 쓴 내용이 어긋나는 상황을 줄이기 위해 만들어졌습니다.

모든 작업은 사용자의 로컬 파일(Obsidian Vault)과 로컬 앱 데이터(포터블 파일과 같은 경로에 존재하는 userData 폴더)를 기준으로 이루어집니다.

배포 파일은 GitHub Releases에서 제공하며, 구체적인 가이드와 자료(플러그인 등)은 [아카라이브 AIOFN 채널](https://arca.live/b/aiofn)에서 제공됩니다.

이런 사람에게 사용을 추천합니다.

- Obsidian을 애용한다.
  - 하지만 Obsidian만으로는 문서 관리에 부족함을 느낀다.
 
- 소설 집필 관련 Obsidian Plugin 중 마음에 드는 게 없다.
  - Obsidian의 구조적인 한계에 봉착해 Obsidian Plugin을 직접 제작할 수 없다.
    - 제작자가 AIOFN을 만들게 된 이유이기도 합니다.

- 문서를 철저하게 분류하고 강박적으로 관리하고 싶다.
  - 문서 Status를 YAML Frontmatter로 관리 및 앱을 통한 직관적인 시각화.
  - Frontmatter 필터링을 지원하는 Sheet, Corkboard 서브 앱을 통한 문서 관리.
    - Sheet : 캐릭터 및 설정 관리에 유용.
    - Corkboard : 플롯 관리에 유용.
    - 회차 관리 : 실제 집필에 유용.

- RAG로 인덱싱한 내 작품을 MCP를 통해 AI에게 학습(모델 학습에 이용되지 않길 바라면 각 공식 홈페이지 또는 Desktop App의 설정에서 데이터 공유 옵션 비활성화)시켜 함께 소설을 만들고 싶다.
  - 제작자도 웹소설 작가이기에, 웹소설을 집필하는 데 AI 사용이 금기시된다는 불문율이 있다는 건 인지하고 있습니다.
  - MCP 도구에 `직접 창작하지 마라`는 제한을 걸어둘 수도 있지만 굳이 그러진 않았습니다.
  - 어디까지나 LLM을 `생성`의 도구가 아니라 `보조`의 도구로 사용해 주시길 바랍니다.
  - 만일 생성의 도구로 사용하셨다가 적발돼 논란에 휘말리셨을 경우, 제작자는 어떠한 책임도 지지 않습니다.
